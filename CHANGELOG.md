## 0.1.0

+ New: Initial release. Hyva theme compatibility companion for `MageMe_EUWithdrawalReceiptVerify` (Pro). Re-renders the public verify page at `/withdraw-contract/verify/index/request_id/<id>/hash/<hex>/` using pure Tailwind utility classes — no LESS, no module-shipped CSS, no jQuery.
+ New: Layout XML `view/frontend/layout/mageme_eu_withdrawal_verify_index.xml` removes the parent module's `<css src="MageMe_EUWithdrawalReceiptVerify::css/verify.css"/>` directive on Hyva theme via `<remove>`. Hyva's Tailwind bundle becomes the sole CSS source.
+ New: Tailwind/Alpine override of the verify-page phtml at `view/frontend/templates/MageMe_EUWithdrawalReceiptVerify/verify/result.phtml` (auto-discovered via `Hyva_CompatModuleFallback`). Same data surface (receipt number, order number, confirmed-at, refund total, item lines, merchant name, integrity hash, SHA-256 monospace block) — only the chrome differs. Hash block uses CSS `select-all` so a click selects the whole hash for system-clipboard copy via Ctrl+C / ⌘+C; no JavaScript copy-button needed (Hyva CSP-strict Alpine forbids inline event handlers, and a pure-Alpine.data registration would add module JS overhead for what `select-all` does natively).
+ New: GDPR Art. 5(1)(c) data minimisation maintained — the consumer's full name is intentionally not rendered (matches the parent module 0.1.2 behaviour and the Stripe/Square/Shopify tokenised-receipt pattern).
